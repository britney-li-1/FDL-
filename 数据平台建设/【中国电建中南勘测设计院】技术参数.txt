4.建设内容 
搭建具备高效数据治理与智能分析能力的数据平台，通过引进成熟、可靠的平台产 品。具备多源异构数据融合引擎，实现对各业务系统、数据库及外部数据源的无缝对接 与统一管理。提供覆盖数据全生命周期的数据开发与分析工具链，涵盖数据采集、存储、 治理、管理及应用等关键环节，确保数据流转安全、合规与高效。同时，结合自动化、智能化的数据质量监控、元数据管理、数据标准化及数据资产管理等功能，形成一套完 整的数据“采-存-治-管-用”全流程应用解决方案，助力中南院实现数据价值的深度挖掘与持续创新。数据平台应满足以下主要组成及能力要求： 
4.1 数据基础平台 
数据基础平台需要构建企业级数据基础设施，为多源异构数据提供分布式存储、湖 仓一体、批流计算、存算一体、多租户管理及智能运维等核心能力，支撑数据全生命周 
期的高效处理。 
1）数据基础平台应全面兼容 Hadoop 生态组件及服务技术，支持 HBase、HDFS、 
Mapreduce、Spark、Flink、Hive、Sqoop、Kafka 等组件的集成与协同运行。 
2）具备湖仓一体数据架构，并结合响应人平台的实际功能，阐述其在数据湖与数 
据仓库融合、统一存储、元数据管理以及数据安全治理等方面的能力。 
3）▲支持 PB 级数据存储和管理能力。 
4）多租户支持，实现租户级资源和数据隔离，并允许租户自助管理。 
5）多项目支持，并支持为每个项目分配单独的资源和权限。 
6）多环境支持，支持开发、测试、生产等环境，并支持配置及数据等资源在多环 
境中进行快速流转或迁移。如响应人通过部署多套数据平台来实现多环境，应支持开发、 
测试环境进行免费试用授权，且支持每隔半年激活试用授权的申请。 
7）▲支持多集群集中管控，可在同一平台上管理多套集群，并能实现集群间的快 
速切换与统一操作，提升跨集群管理效率。 
8）支持集群弹性伸缩，可在不停机情况下完成平台扩容与升级，包括集群规模扩 
展及功能组件角色新增，满足业务增长需求。 
9）支持通过可视化界面实现集群全量大数据组件的常规运维操作，包括组件服务 
及实例的启动、停止、重启、扩容、缩容等，简化运维流程。 
10）支持异常节点快速恢复，可通过停止、启动、重启、滚动重启等操作恢复节点 
服务，保障集群稳定性。 
6111）支持对集群、节点服务器、组件、中间件、数据库等资源进行实时监控、并支 
持对所有监控异常项预设告警策略，支持通过邮箱、Webhook 等方式发送告警信息。 
12）支持自动化运维脚本及批量操作，提升日常运维管理效率，减少人工操作失误。 
13）具备完善的日志管理及审计功能，能够对平台各类操作、数据访问、系统事件 
进行全流程记录，支持日志的集中存储、检索与分析，以满足合规性与安全管理要求。 
14）支持多种身份认证与权限管理机制，如 LDAP、OAuth 等，实现用户身份统一认 
证与细粒度权限控制，保障平台安全性。 
15）平台应支持高可用架构设计，具备主备切换、故障自动转移等能力，确保关键 
服务的持续稳定运行。 
16）支持资源配额与用量统计分析，实时监控各租户和项目的资源使用情况，便于 
合理分配与成本管控。 
4.2 结构化数据平台 
数据治理与管理平台须具备覆盖数据全生命周期的核心能力：可采集结构化、半结 
构化、非结构化数据，支持多方式采集，结合“可视化+SQL”开发与全流程调度运维， 
实现数据高效汇聚加工。同时，需建立数据标准、指标、模型体系，搭配全流程数据质 
量管控与元数据管理，保障数据一致、准确、可扩展；构建数据安全防护体系，通过分 
级分类、权限管控等确保合规使用，打造资产视图实现资产可视化管理。以数据服务市 
场提供 API 管理，支持数据服务筛选、申请与审批，实现数据价值快速转化。 
4.2.1 数据采集 
实现结构化、半结构化、非结构化数据的统一采集汇聚与任务调度，支持关系型数 
据库、大数据平台、空间数据、文本数据、实时数据、填报数据等各类数据源，并通过 
统一数据填报工具实现无业务系统时的数据快速采集，解决数据来源问题。 
1）▲提供统一数据源管理功能，支持通过可视化页面对内外部各类结构化、半结 
构化和非结构化数据源进行集中维护与管理。支持的数据源类型包括但不限于：数据库 
（如达梦、金仓、GaussDB、GBase、OpenGauss、OceanBase、Greenplum、SqlServer、 
MySQL、Oracle、ClickHouse 等）、文件（如本地文件、FTP/SFTP、S3 等）、大数据库 
平台（Hadoop、MaxCompute 等）、接口（HTTP、Kafka 消息队列等）。未支持的主流数 
据源，响应人应承诺免费进行适配。 
2）▲应支持 SqlServer、MySQL、Oracle、PostgreSQL、金仓、达梦等数据库的变 
更数据捕获能力，若不支持金仓、达梦数据库的变更数据捕获，应给出具体解决方案。 
623）支持通过可视化配置式的方式进行数据全量和增量采集、处理和存储。 
4）支持全库同步，且支持数据库表结构同步。 
5）应具备丰富的数据采集、加工、清洗、映射、转换、脱敏等处理的组件，且支 
持组件扩展。 
6）▲应具备接入外部 AI 的能力来赋能 ETL 过程，例如：支持调用图片 OCR 的接口 
来进行图片内容识别。 
7）应支持 Excel、CSV、JSON 等格式数据的导入操作。 
8）应具备配置可视化表单方式来进行数据填报，且具备流程审批功能。 
9）具备采集任务的调度管理和监控功能，支持任务的定时执行、失败重试及运行 
状态告警，便于运维人员及时发现并处理采集异常。 
10）支持采集元数据的自动登记与管理，自动记录采集任务的来源、目标、时间、 
规则等信息，便于后续追溯与审计。 
11）支持多租户环境下的数据源隔离与权限管控，确保不同租户或用户的数据采集 
任务不会相互影响，并能灵活配置访问权限。 
12）具备对采集数据进行加密传输与存储的能力，保障数据在传输和落地过程中的 
安全性，满足数据安全合规要求。 
13）支持采集任务的版本管理，能够记录和回溯采集任务的配置变更历史，便于任 
务的维护和升级。 
4.2.2 数据加工与计算 
提供强大计算与开发引擎，通过可视化编排设计器，利用封装算子实现实时与离线 
数据处理工作流任务的设计与调度，满足复杂场景数据加工与计算需求。 
4.2.2.1 离线开发 
1）▲支持通过 DAG 的可视化开发，支持流批一体的数据开发功能，支持分层模 
型的定义，血缘关系的自动维护，支持基于算子的 DAG 图形定义和仿真计算；算子支 
持常用数据处理组件和数据挖掘，例如：数据清洗、拆分列、派生列、行转列、列转 
行、JOIN、UNION、列选择、数据去重、值替换、排序、聚合、过滤、增加序列号、SQL 
脚本、Python 脚本组件、决策树、随机森林、线性回归等。 
2）具备数据融合分析能力，能够提供联合查询能力和多种异构数据源的即席查询 
能力。 
633）支持任务模板管理，用户可保存常用任务为模板，便于快速复用与批量创建， 
提高开发效率。 
4）具备任务运行状态监控功能，能够实时展示各类开发任务的执行进度、日志信 
息和异常告警，方便运维人员及时跟踪和处理。 
5）支持任务版本管理，记录每次任务配置和代码的变更历史，支持版本回溯与恢 
复，保障任务开发的可控性和安全性。 
6）支持任务的定时调度和依赖管理，灵活配置任务间的执行顺序与触发条件，满 
足复杂数据处理流程的编排需求。 
7）▲具备丰富的算子库和开发组件，支持通过 SQL、JAR、Python 等方式扩展与自 
定义，便于满足多样化的数据处理与分析需求。 
4.2.2.2 实时开发 
1）▲支持最新版本的 Fink CDC 所支持的主流数据库，支持 Kafka 数据源；支持通 
过拖拽式或者脚本的方式完成 Flink 任务的设计和开发。 
2）具备实时清洗能力，支持清洗过滤、转换功能，具备内置组件，包括但不限于： 
过滤组件（长度过滤、正则过滤、精度标准化过滤、比较过滤等）、清洗组件（全角半 
角转换、字典标准化、日期标准化、字符串截取等）；支持自定义清洗规则，支持配置 
清洗中问题数据处理方式。 
3）支持高吞吐量和低延迟的数据处理，能够满足大规模实时数据流的处理需求， 
保障数据处理的及时性和稳定性。 
4）具备完善的任务监控与告警机制，能够实时跟踪 Flink 任务的运行状态、性能 
指标和异常情况，便于运维人员进行问题定位和快速响应。 
5）支持任务的灵活调度与动态扩展，能够根据业务需求实现任务的自动扩容、缩 
容及重启，提升系统的可用性与弹性。 
6）支持结果数据的多样化输出方式，包括但不限于写入主流数据库、消息队列、 
文件系统等，满足不同业务场景的数据消费需求。 
7）具备丰富的算子扩展能力，支持自定义开发 Flink 算子，满足复杂业务逻辑和 
个性化数据处理需求。 
4.2.2.3 任务调度 
1）具备调度管理功能，包括一次性调度与周期性调度，其中周期性调度应支持按 
64分钟、小时、日、周、月、年调度，工作流可跨周期依赖。 
2）支持灵活的依赖配置，允许任务间进行前置依赖、后置依赖及条件依赖设置， 
确保各类复杂业务流程顺利执行。 
3）具有可视化调度界面，支持通过拖拽方式设计和管理调度流程，便于用户直观 
操作和维护。 
4）提供丰富的调度策略与失败重试机制，可根据实际业务需求配置重试次数、重 
试间隔及异常处理方式，提升任务调度的健壮性。 
5）支持调度任务的动态扩展与缩减，能够根据资源使用情况自动分配计算资源， 
实现高效资源利用。 
6）具备详尽的调度日志与审计功能，能追溯任务执行历史，便于问题定位与合规 
审查。 
4.2.3 数据治理 
基于元数据管理、主数据管理、数据标准管理、数据模型管理、数据质量管理、数 
据安全管理、指标管理、数据资产管理等模块，解决数据追踪溯源、主数据管理、数据 
定义统一、数据建模、数据质量、数据安全、指标生命周期管理、数据资产等问题，提 
升数据管理水平与可用性。 
4.2.3.1 元数据管理 
元数据管理旨在对企业内各类数据的描述信息进行统一管理和维护。通过元数据管 
理，可以实现对数据来源、结构、变更、使用及流转过程的全面追踪和分析，有效提升 
数据的可发现性、可追溯性及安全性。 
1）元数据管理应支持自动通过数据源获取元数据模型。可设置自动采集和手工采 
集，支持全量采集、手动补录采集，且具备详尽采集日志记录。采集后元数据可生成数 
据地图，支持自动绘制数据地图和手动维护数据地图，地图能够下钻，并可直接对数据 
地图上元数据进行分析。支持相关血缘、影响分析及变更管理的监控与通知，让数据管 
理者能更方便快捷的发现数据问题。 
2）▲支持多数据源异构元数据采集，能够对关系型数据库、非关系型数据库、大 
数据平台、云数据服务等多种数据源进行元数据采集，提升元数据管理的广度和深度。 
3）支持元数据的增删改查，元数据版本发布、版本对比，及元数据导出，并支持 
查看该元数据所关联的数据标准、数据质量规则。 
654）具备元数据权限控制与安全管理功能，可以对不同用户或角色分配元数据访问、 
编辑、发布等权限，保障企业数据安全与合规性。 
5）支持元数据的自动同步与更新机制，能够根据数据源结构变更自动触发元数据 
模型的同步更新，确保元数据的时效性和准确性。 
6）提供元数据可视化分析工具，支持对元数据分布、变更趋势、使用频率等进行 
可视化展示，便于管理者直观了解元数据现状，辅助决策。 
7）支持元数据与业务流程、数据资产的关联管理，实现元数据与企业核心业务之 
间的映射与追踪，提升数据价值挖掘能力。 
4.2.3.2 数据标准管理 
数据标准管理旨在通过制定统一的数据标准和规范，确保企业内部各类数据在采集、 
存储、交换和应用过程中的一致性与规范性。通过数据标准管理，可以有效推动数据定 
义的统一，提升数据的互操作性和可用性，降低数据管理的复杂度，为数据质量提升和 
数据安全保障奠定坚实基础。 
1）数据标准管理应支持标准文件全生命周期在线管理，功能包括但不限于上传、 
发布、下载、搜索、查看、预览及分类归档等操作，实现标准文件的集中化存储与快速 
检索。 
2）▲支持数据标准的版本管理，能够对不同版本的数据标准进行发布、对比、回 
溯和历史记录查看，确保标准的演进过程透明可控。 
3）▲支持数据标准集属性定义，标准集可选属性，包括但不限于：业务属性、技 
术属性、管理属性、质量属性、生命周期属性。支持在系统中建立数据标准，支持标准 
excel 导入导出，支持从元数据拾取批量新建标准。 
4）具备数据标准的权限管理功能，可根据用户角色分配数据标准的访问、编辑、 
发布等权限，保障标准文件的安全性和合规性。 
5）支持数据标准与元数据、数据质量规则的关联管理，实现数据标准与实际数据 
资产、业务流程的有效映射，提升标准化管理的应用价值。 
6）提供数据标准的变更通知和自动同步机制，能够在标准内容发生变化时及时通 
知相关人员，并自动同步到相关系统或业务流程，确保标准的实时性和一致性。 
7）支持数据标准的可视化展示与分析，能够对数据标准的分布、应用范围、变更 
趋势等进行图表化展示，辅助管理者进行决策和优化。 
664.2.3.3 数据质量管理 
通过建立完善的数据质量管理体系，能够及时发现并解决数据中的各类问题，确保 
数据在采集、存储、传输、分析等各环节的高质量流转。 
1）支持通过配置各种规则检测数据的完整性、规范性、一致性、准确性、唯一性、 
时效性。 
2）支持通过制定的数据标准，自动评估当前数据质量。 
3）▲支持对数据质量问题进行识别、度量、监控和预警。能够周期性自动出具数 
据质量报告，报告中能准确说明数据存在的问题，并将问题数据明细展示出来，用户可 
以查看与下载数据质量报告，从而提升数据管理水平。 
4）▲支持基于数据源头和数仓两个层级的数据质量的稽核和质量改进，并按照 
质量的评定结果选择清洗方案，再将相应的整改任务自动或手动形成工单分配给相应 
的整改人 
5）支持与元数据管理系统的集成，实现数据质量规则与数据资产的动态关联，提 
升数据治理的协同性。 
6）具备数据质量指标的自定义和分级管理能力，能够满足不同业务场景下的质量 
评估需求。 
7）支持数据质量问题的分类归档和统计分析，辅助企业进行问题溯源和持续优化。 
8）提供数据质量提升建议和整改跟踪机制，帮助相关责任人及时修复问题并验证 
整改效果。 
4.2.3.4 数据安全管理 
通过制定科学的数据安全策略和措施，可以有效防止数据泄露、滥用和非法访问， 
确保企业核心数据资产的机密性、完整性和可用性。数据安全管理应有助于提升企业合 
规能力，满足相关法律法规的要求，为企业的数字化转型和业务创新提供坚实的安全保 
障。 
1）提供精细化数据访问控制功能，支持用户针对库、表、行、列（字段级）、目 
录等不同层级申请读写权限，支持配置权限有效期，并可对库、表权限进行灵活的授权 
与收回操作，实现权限的精准化管理。 
2）▲支持数据分类分级、数据加密、数据脱敏、数据的分权分域管理及租户、项 
目隔离。 
673）▲能够通过智能化方案进行数据的分类分级设置，同时支持手工分类分级的 
修正，支持静态或动态的脱敏功能，支持的脱敏策略包括掩码、哈希、加密、截断等 
多种脱敏类型。支持国密 SM3/SM4、MD5、AES、DES 等数据加密算法。 
4）支持行为审计能力，平台自动记录表、敏感列的访问记录，包括访问数据的用 
户名、时间、 执行的 SQL、读取行数等关键信息，实现操作行为的可追溯。 
5）支持安全策略的灵活配置，能够根据企业不同业务场景设定黑白名单、异常访 
问自动告警，并对高风险操作进行实时阻断，提升整体数据安全防护能力。 
6）具备权限变更审计能力，确保权限授予和回收过程有据可查，提升权限管理的 
合规性和透明度。 
4.2.3.5 数据模型管理 
通过科学的模型设计与管理，实现数据资产的结构化、标准化和高效流通。应提供 
完善的数据模型管理功能，能够提升数据的组织与表达能力，还能为数据分析、业务决 
策和系统集成提供坚实基础。 
1）具备在线建模功能；能够以在线的方式管理逻辑模型和物理模型，并支持对不 
同模型基于主题域实现分组、编目管理。 
2）▲具备逻辑模型和物理模型的联动功能，能够实现增量更新；支持物理模型一 
键建表。 
3）具备可视化建模功能，具备数据模型管理、模型加工任务管理功能，能够自动 
生成数据加工任务，配置任务加工方式包括不限于多表联合、多表连接、自定义映射、 
多表去重。 
4）具备批量导入和导出逻辑模型功能。 
5）支持模型版本管理，能够对模型的历史版本进行保存、比对与回溯，便于模型 
的演进和变更追踪。 
6）支持模型权限管理，可对不同用户或用户组分配模型的查看、编辑、发布等操 
作权限，确保模型管理的安全与合规。 
7）具备模型质量校验功能，能够对模型的结构、字段类型、命名规范等进行自动 
校验，及时发现和预警潜在问题。 
8）▲支持模型与数据血缘分析，能够清晰展现模型与数据源、数据表、字段之间 
的血缘关系，便于追溯数据流向。 
689）支持模型文档自动生成，能够依据模型结构、字段信息等内容自动生成文档， 
提升模型管理的规范性与可读性。 
4.2.3.6 数据指标管理 
数据指标管理应规范指标的定义、计算和应用过程，提升数据分析的标准化与透明度。通过科学的指标管理，企业能够统一数据口径，确保各类报表、分析和业务决策的 数据基础一致，有效避免“各说各话”的现象。同时，指标管理为业务部门提供灵活、可复用的分析工具，助力企业实现精细化运营和持续优化。 
1）▲支持定义原子指标、衍生原子指标、派生指标、复合指标，可以根据原子指 
标生成衍生指标，方便指标的计算调用，指标可以直接供报表调用。 
2）▲提供可视化指标开发功能，支持通过设计器点选配置原子指标、复合指标、 
派生指标的计算规则。支持常见函数表达式，涵盖 SUM、MIN、MAX、COUNT、DISTINCT、 
AVG 等聚合函数。支持将跨表关联字段作为维度，实现维表与事实表的跨表计算构建指 
标等功能。 
3）▲提供指标映射关联功能，支持指标与维度、数据源、数据表、字段等进行精 
准映射和关联，可基于映射结果查看采样数据，确保指标数据来源的透明化。 
4）支持指标订阅与自动化取数，用户可将多维度数据分析结果保存为周期性取数 
任务，实现数据的定时抽取、存储及查看，确保数据的时效性与可复用性。 
5）▲提供指标血缘分析功能，支持对指标的数据源、计算逻辑、关联字段等血缘 
关系进行全链路追溯，清晰呈现指标的生成路径。 
6）具备单个指标依历史详情记录功能。 
7）支持指标权限管理，能够针对不同用户或用户组分配指标的查看、编辑、发布 
等操作权限，确保指标管理的安全性与合规性。 
8）具备指标质量检测和预警功能，能够自动校验指标的定义规范、计算逻辑和数 
据一致性，及时发现并提示潜在异常或错误。 
9）支持指标文档自动生成，根据指标的结构、计算规则、来源等信息自动生成标 
准化文档，提升指标管理的规范性和可读性。 
10）具备指标生命周期管理能力，能够对指标的创建、变更、废弃等全流程进行记 
录和管控，便于指标体系的持续优化和演进。 
11）具备不低于十万级指标管理的能力。 
694.2.3.7 标签管理 
标签管理旨在通过标准化、系统化的标签体系，为企业数据赋予多维属性和语义， 
实现数据的高效分类、精准检索与智能分析。通过灵活的标签定义与管理，企业能够提 
升数据资产的可用性和可扩展性，支持业务场景的快速变化和多样化需求。 
1）支持标签体系可视化构建，可通过界面自定义标签分类、标签属性及关联规则， 
支持配置标签层次结构与权限管理，满足数据多样化分类与精细化管控需求。 
2）支持标签使用监控与分析，可实时统计标签调用频次、关联数据资产数量等。 
3）支持标签生命周期管理，能够对标签的创建、变更、废弃等操作进行全流程记 
录，便于标签体系的持续优化和演进。 
4）支持标签文档自动生成，根据标签的结构、定义、应用场景等信息自动生成标 
准化文档，提升标签管理的规范性和可读性。 
5）支持大规模标签管理能力，能够满足十万级以上标签的高效管理和运维需求。 
4.2.3.8 数据资产管理 
数据资产管理旨在对企业内各类数据资源进行统一归集、规范管理和高效利用。通 
过建立完善的数据资产目录、资源挂接、可视化预览及全量检索等功能体系，能够提升 
数据的发现、访问和协同效率，确保数据资产安全与合规流通。同时，数据资产管理应 
有助于挖掘数据潜在价值，支撑企业业务创新与智能决策，实现数据驱动的持续成长。 
1）提供资产目录灵活定义功能，支持通过可视化界面进行目录结构的自定义配置， 
也可基于现有数据资产批量导入生成目录，满足不同场景下的目录构建需求。 
2）提供资源挂接全流程管理功能，支持通过可视化页面完成库表、文件、接口等 
各类数据资源的定义、属性配置及目录挂接操作，实现数据资源与目录的精准关联。 
3）提供数据资源可视化预览功能，支持在可视化界面中直接查看已挂接数据资源 
的样本数据、结构信息等，帮助用户快速了解数据内容。 
4）▲支持全量数据资产统一检索功能，可对数据库、表、指标、标签、API 接口等 
资产进行多维度检索，支持关键词模糊匹配及精确查询，提升检索效率。 
5）提供资产目录管理功能，支持通过设置目录的共享范围、开放类型对外发布， 
涵盖目录发 布申请、发布审核、版本管理等全流程审批管控。 
6）具备数据资产分析统计功能，提供数据资产相关的数据库信息，实现统计数据 
资产按照不同类型的资产个数，数据量大小，资产个数及数据量变化情况，资产在业务库中的分布情况等。 
7）支持数据资产安全与合规管理，能够对数据资产的访问、使用、共享进行权限 
配置和合规性审查，确保数据资产在流通和使用过程中符合相关法规和企业政策。 
8）提供目录变更历史追溯能力，支持查看目录结构及资源关联关系的历史变更记 
录，保障数据资产管理的透明性和可控性。 
9）支持资产目录与标签体系的深度集成，实现目录与标签的双向关联，提升数据 
资产的多维检索及智能分类能力。 
10）支持目录自动生成文档，基于资产目录结构、资源属性及挂接情况，自动生成 
目录说明文档，提升目录管理的规范性与可读性。 
11）支持大规模目录管理能力，能够高效管理数万级以上的数据资产目录，保障系 
统稳定运行和运维便捷性。 
4.2.4 数据服务 
数据服务旨在为企业各类业务场景提供高效、可靠的数据访问与应用能力。通过统 
一的数据服务平台，能够实现数据的标准化接口发布、权限管控、服务编排与监控分析， 
支持多种数据访问方式（如 API、批量导出、实时订阅等），满足业务系统、分析平台、 
第三方应用等多元化的数据需求。 
1）支持可视化方式创建数据服务，包括但不仅限于数据库表、数据资产与数据指 
标快速生成数据服务。支持对数据服务进行安全认证和授权管理。 
2）▲支持对接口进行编排管理，包括对接口进行编辑、重组、排序操作。具备 HTTP 
自定义服务能力，具备对服务请求报文和应答报文的转换。 
3）支持注册外部 API 能力，支持手工录入和导入方式进行 API 服务配置。 
4）▲支持接口的安全策略设置，包括内外网安全隔离，IP 黑白名单设置；支持接 
口的熔断、限流、协议转换功能。 
5）支持数据订阅和推送功能，当数据发生变化时，数据技术平台能够及时将数据 
推送给订阅者。数据订阅和推送功能可以提高数据的实时性和共享性。 
6）▲提供数据服务使用统计分析功能，支持多维度查看数据服务调用情况，可按 
时间维度展示调用次数、错误次数、最长响应时间、平均响应时间等核心指标的统计数 
据，并以图表形式直观呈现。 
7）提供数据服务智能报警管理功能，支持自定义创建数据服务 API 的报警策略， 
涵盖 QPS（每秒查询率）、错误率、平均响应时间、慢查询占比、慢查询数量等关键指 
71标的报警规则配置，可根据实际需求自定义报警阈值和通知方式等。 
8）支持服务生命周期管理，包括服务的发布、下线、变更、版本控制等全流程管 
理，保障数据服务的可追溯性和管理规范性。 
9）支持服务文档自动生成和维护，能够根据数据服务接口定义自动生成接口文档， 
提升开发者使用和运维的便捷性。 
10）支持服务性能监控和健康检查，实时监控服务运行状态，及时发现和预警异常， 
保障数据服务的稳定运行。 
11）支持多租户和多环境隔离管理，满足不同部门、项目的数据服务资源独立管理 
和访问安全需求。 
4.2.5 数据分析与应用 
响应人需协助采购人与待选定的数据可视化平台和报表系统对接，具体协助内容包 
括但不仅限于： 
1）协助对数据可视化平台进行需求调研，梳理双方接口、数据格式、访问方式等 
技术细节，形成对接方案。 
2）协助与数据可视化平台进行数据接口联调，确保数据能够按照约定格式和周期 
稳定传输。 
3）协助进行数据权限、安全机制的配置，保障对接过程中的数据安全和合规性。 
4）配合采购人完成对接过程中的功能测试、性能测试以及问题修复，确保数据传 
输的准确性和实时性。 
5）负责提供对接相关的技术文档。 
4.3 非结构化数据能力 
4.3.1 数据采集与入湖 
1）▲多源异构数据入湖：支持增量、批量、实时流式等模式，将本地文件、 
FTP/SFTP 、HTTP/HTTPS 文件下载接口、对象存储、S3、Excel、CSV、JSON 等多种非结 
构化数据接入湖仓一体平台的统一存储层。 
2）提取文件通用元数据（如格式、大小、来源、时间）并写入到统一的元数据 
仓库中。 
4.3.2 数据存储和管理 
1）▲采用湖仓一体化架构，支持海量非结构化数据的安全存储、备份与恢复，并 
72实现灵活的数据管理和高效数据访问。 
2）具备分布式存储能力，能够横向扩展以满足数据量持续增长的需求，支持多节 
点冗余与容错，保障数据的高可用性与业务连续性。 
3）具备完善的数据安全机制，包括数据加密和访问权限控制。 
4.3.3 数据解析与处理 
1）▲支持灵活调用外部 AI 的能力来赋能数据解析和元数据入库，例如：支持调用 
外部图片 OCR 接口来进行图片内容识别。 
2）支持对非结构化数据标签化，提升数据的可用性和检索效率。 
4.3.4 数据资产管理 
1）▲支持将非结构数据与结构化数据，作为企业数据资产总目录集中管理，且能 
快速发布为服务，供用户浏览与使用申请。 
2）具备元数据管理能力，对非结构化数据进行统一的元数据采集、存储和维护， 
提升数据的可检索性与可管理性。 
3）具备数据生命周期管理能力，支持灵活的数据权限与访问控制。
