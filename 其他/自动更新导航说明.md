# 🔄 自动更新导航页面

## 📋 说明

**导航页面不会自动更新**，但你可以使用脚本快速更新！

### ⚠️ 重要：什么时候需要更新导航？

| 操作 | 是否需要更新导航？ | 是否需要推送？ |
|------|------------------|---------------|
| **新增 HTML 文件** | ✅ **需要** | ✅ 需要 |
| **删除 HTML 文件** | ✅ **需要** | ✅ 需要 |
| **修改文件内容**（不改名） | ❌ **不需要** | ✅ 需要推送文件本身 |
| **重命名文件** | ✅ **需要** | ✅ 需要 |

**总结**：
- 只有**文件列表变化**（新增/删除/重命名）时才需要更新导航
- 只是**修改文件内容**时，导航页面不需要更新，但文件本身需要推送

---

## 🚀 使用方法

### 场景 1：新增或删除文件（需要更新导航）

**步骤 1**：创建或删除 HTML 文件

**步骤 2**：运行脚本更新导航
```bash
cd "/Users/qiushengming/Desktop/李慧雯设计素材/cursor test"
./生成导航页面.sh
```

**步骤 3**：提交并推送
```bash
# 添加所有更改（包括新文件和导航页面）
git add .
git commit -m "添加新页面并更新导航"
git push
```

---

### 场景 2：只修改文件内容（不需要更新导航）

**步骤 1**：修改 HTML 文件内容

**步骤 2**：直接提交并推送（不需要运行脚本）
```bash
git add 修改的文件.html
git commit -m "更新页面内容"
git push
```

**导航页面不需要更新**，因为文件列表没有变化。

---

## ✨ 脚本功能

- ✅ 自动扫描所有 HTML 文件
- ✅ 按目录结构分类显示
- ✅ 自动识别分类图标（⏰ 定时管道、⚡ 实时管道、🔍 检测任务等）
- ✅ 排除导航页面本身
- ✅ 显示文件路径和标题

---

## 📝 示例

### 示例 1：新增文件
假设你创建了一个新文件 `新功能测试.html`：

1. 运行脚本：`./生成导航页面.sh` ✅
2. 脚本会自动将新文件添加到导航
3. 提交并推送：`git add . && git commit -m "添加新功能测试页面" && git push`

### 示例 2：修改文件内容
假设你修改了 `index.html` 的内容：

1. **不需要**运行脚本 ❌
2. 直接推送：`git add index.html && git commit -m "更新主页内容" && git push`
3. 导航页面保持不变（因为文件列表没变）

### 示例 3：删除文件
假设你删除了 `旧页面.html`：

1. 运行脚本：`./生成导航页面.sh` ✅
2. 脚本会自动从导航中移除该文件
3. 提交并推送：`git add . && git commit -m "删除旧页面" && git push`

---

## 💡 提示

- 脚本会保留原有的样式和布局
- 新文件会自动按目录分类
- 根目录的 HTML 文件会显示在"主页"分类
- 子目录的文件会显示在对应的分类下
